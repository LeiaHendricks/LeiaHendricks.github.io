FreeTherapy cpt
Case Study

Overview

Cape Town underprivileged communities need a way to access
free mental health resources. The product needs to be designed
for mobile first and then branch to an accompanying website.

Goal

To create an app and website for FreeTherapy cpt so that
underprivileged communities can have access to mental health
resources. Accessibility features such as language translation
will be available as there are 11 official languages.

Role

Lead UX Designer

Tools

Figma

Team

Solo (Course Project)

Timeline

Nov - Dec 2023

Pain Points

Booking Process

Should be short and to the point. Suggesting counselors based on their reason for booking can sped up the process.

Accessibility

Trying to navigate a product when you are not confident in english can be frustrating so there should be a language translate feature.

Accessibility

The website should offer a range of languages since it would be based in Cape Town.

Personas: Amy

Problem statement:
Amy is an unemployed high school student who needs access to free mental health resources because she does not have the money to afford a counselor.

User Journeys

The process to book a session should be short. The user should not have to pay for the session as the product would be free. The user would also not have to make an account as it will be linked to their South African ID number and phone number.

Sketches

The goal for this section was to start with the mobile app and once all of the paper wireframes were finished, I could move onto designing what the website would look like.

My goal for the desktop size was to look similar to the app utilising more space. Both the app and website should have the same features so that the experience can be the same on both formats.

Wireframes

The home screen was designed in a way that everything could be found easily. The main user flow of booking a session can be started in several areas such as the navigation bar and further down the page so that the user doesn’t have to search to find it.

Low-fidelity prototype

Using the completed set of digital wireframes, I created a low-fidelity prototype. The primary user flow i connected was ordering a drink of choice so that the prototype could be used in a usability study.

Link to Low-Fidelity prototype

Usability Studies

I conducted two rounds of unmoderated usability studies. Findings from the first study helped guide the designs from wireframes to mockups. The second study used a high-fidelity prototype and revealed what aspects of the mockups needed refining.

Round 1 findings

  • Users need a more intuitive and shorter way to change the app language.
  • The labels in the app should be simple so that users immediately know what they do.
  • All payment options were not considered when building the app prototype.

Round 2 findings

  • There should be a landing page added where users can either log in or create an account.
  • Some screens should be changed to all have the same colour scheme.
