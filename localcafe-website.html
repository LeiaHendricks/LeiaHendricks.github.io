LocalCafe Website
Case Study

The app allows customers to earn points with the
purchases of their beverages.

Overview

LocalCafe needs a website with an ordering process as well as a
way to learn more about the franchise so that they can bring in
more customers.

Goal

To create a website for LocalCafe that is on par with competitors
in order to bring in more customers and increase customer
retention. Accessibility features such as language translation will
be available.

Role

Lead UX Designer

Tools

AdobeXD

Team

Solo (Course Project)

Timeline

Sept - Oct 2023

Ordering Process

Placing large orders in person can be frustrating when there are too many things to remember. Placing large orders through the website should be made simple.

Time

Waiting in line to place orders can be time consuming. Customers should be able to place orders ahead of time so that they are able skip the line.

Accessibility

Trying to place orders in person when you are not confident in English can be frustrating so there should be an website translate feature.

Accessibility

There are limited translation features so the website should offer a range of languages.

Personas: Zareen

Problem statement:
Zareen is a busy intern who needs an easy way to submit and track orders and be notified when they are ready because it takes too much time to stand in line and wait for the orders to complete.

Personas: Joseph

Problem statement:
Joseph is a non-native language speaker who needs more translation options because he struggles to understand baristas who speak English too fast.

User Journeys

The ordering process should be a simple and efficient way to place orders no matter the size. The app should be simple enough that someone who does not understand English can navigate to change the app to their desired language.

Sitemap

I wanted the main part of the website to be a simple matrix sitemap but the order process to be sequential so that the user had an easy path to follow. Learning from the usability studies on the loyalty app, I made the language settings an easily accessible dropdown menu on the home screen.

Sketches

The goal for this section was to try and get an idea for the website layout and what the homescreen was envisioned to look like. I prioritised a simple layout so that users could understand the features at a glance.

My goal for the mobile screen sizes was to look similar to the app but missing some features so that the user would have incentive to download the app (eg. the rewards program)

Wireframes

Feedback was vital in this section as there was a lot of uncertainty in the beginning. I learnt from previous feedback that the language settings needed to be easy to find so I put them in the navigation pane.

mobile variant:
To save space I moved all navigation and footer information to a hamburger menu on the side and mimicked the design of the mobile app keeping the order button on the top of the screen for easy access.

Low-fidelity prototype

Using the completed set of digital wireframes, I created a low-fidelity prototype. The primary user flow I connected was ordering a drink/snack of choice so that the prototype could be used in a usability study.

Usability Studies

I conducted an unmoderated usabilty study for 15 - 20 minutes on 5 participants of various backgrounds and these were the findings:

Order Method

Users want a more intuitive way to start the order process.

Order Process

The order process should be as short as possible.

Layout

Users don’t like when they can see the header and the footer at the same time.
